1. Pre-Processing
   * 관리자는 도착지, 도착시간을 유저에게 할당한다.
   * User 1은 자신에게 할당된 event를 확인하고, 그 event에 대한 출발지, 예상준비시간을 입력한다.
   * 시스템은 카카오맵 API를 이용하여 출발지에서 도착지까지의 이동소요시간을 계산하여, 예상출발시각과 준비시작시각을 역산한다.
      * 예상출발시각 = 도착시각 - 이동소요시간
      * 준비시작시각 = 예상출발시각 - 예상준비시간
   * 시스템은 준비시작시각 10분 전, 5분 전, 준비시작시각에 User 1에게 준비시작알림을 보낸다.
   * 시스템은 예상출발시각 10분 전, 5분 전, 예상출발시각에 User 1에게 출발권고알림을 보낸다.
2. Event
   * 시스템은 User 1이 예상출발시각 10분 전, 출발지 5m 이내에 존재하는지 확인한다.
      * 존재하지 않는다면 출발지를 현위치로 수정한다.
      * 출발지에 존재한다는 “isLocated” = true 로 수정하고, 예상 도착 시각을 계산한다.
   * 시스템은 출발지에서 5m를 벗어나면 출발로 간주하고, 실제 출발 시각을 업데이트한다.
   * 시스템은 User 1에게 이동 교통수단과 함께 이동소요시간을 안내한다.
   * 시스템은 도착지에서 50m 이내에 User 1이 들어오면 관리자에게 “User 1 도착 임박”이라는 알림을 보낸다. “arriveSoon=true”
   * 시스템은 도착시각 이전 User 1이 도착지에 들어오면 출석 상태를 “Present”로 수정한다.
   * 시스템은 도착시각에 User 1의 출석 상태를 확인하고, “Present”가 아닌 경우 “Absent”로 수정한다.
   * 시스템은 도착시각~10분에 User 1이 도착지에 들어오면 출석 상태를 “Present”로 수정한다.
   * 시스템은 도착시각+10분~도착시각+30분에 User 1이 도착지에 들어오면 출석 상태를 “Late”로 수정한다.
   * 시스템은 도착시각+10분이 지났을 때 User 1이 arriveSoon==false인 경우, User 1에게 상황별 맞춤 문자를 보낸다. 


다음의 프로세스를 google sheet를 데이터베이스로 사용하고, google sheet 내부의 App script를 이용하여 백엔드 데이터를 처리하고, 웹앱으로 프론트 처리해서 구현해. 현재 유저의 위치는 핸드폰 자체에서 구글이 위치 정보를 얻을 수 있는 걸 이용해서 얻도록 한다.

교통 정보는 TMapAPI를 이용하며, 현재 작성되어 있는 코드를 기반으로 프로젝트 전체 구성을 파악한 후, 작업 설계를 항상 먼저 고안한 후 이용자에게 허락을 구하고, 그 후 수정을 진행하도록 한다.
